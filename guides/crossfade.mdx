A crossfade transition stops one animation and starts another at the same time, using a shared fade duration for both operations. This keeps the character's motion fluid during the switch — rather than a hard cut or a brief unanimated gap, you get a smooth overlap where both tracks blend together momentarily.

## How Crossfade Works

`:crossFade()` calls `:stop()` on the `fromId` animation and `:play()` on the `toId` animation simultaneously, using the same `time` value for both the fade-out and fade-in.

```lua
AnimationService:crossFade(model, fromId, toId, time) -> AnimationTrack
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `model` | `Model` | The character model |
| `fromId` | `string` | Key of the animation to fade out |
| `toId` | `string` | Key of the animation to fade in |
| `time` | `number` | Fade duration in seconds (applied to both directions) |

`:crossFade()` returns the new `AnimationTrack` for `toId`.

## Basic Usage

```lua
-- Transition from Idle to Run over 0.3 seconds
Anim:crossFade(char, "Idle", "Run", 0.3)
```

## Crossfade vs. Manual Stop + Play

You can achieve the same result manually, but `:crossFade()` is more concise and keeps both timings in sync automatically.

<CodeGroup>
```lua Manual approach (less smooth)
-- Manual approach — easy to let the fade values drift out of sync
Anim:stop(char, "Idle", 0.3)
Anim:play(char, "Run", "Movement", true, 0.3)
```

```lua crossFade (cleaner)
-- crossFade ensures both sides use the same duration
Anim:crossFade(char, "Idle", "Run", 0.3)
```
</CodeGroup>

<Note>
  `:crossFade()` returns the new animation track (`toId`). Store it if you need to adjust the track's speed, weight, or connect events directly.

  ```lua
  local runTrack = Anim:crossFade(char, "Idle", "Run", 0.25)
  runTrack:AdjustSpeed(1.5)
  ```
</Note>

## Choosing a Fade Duration

The `time` value controls how natural the transition feels. Too short and the cut feels abrupt; too long and the blend lingers past the point where it looks intentional.

| Scenario | Recommended `time` |
|----------|-------------------|
| Locomotion (idle ↔ walk ↔ run) | `0.2` – `0.3` s |
| Action to movement (attack → idle) | `0.15` – `0.25` s |
| Dramatic slow transitions | `0.4` – `0.6` s |
| Instant snap (no blend) | `0` |

<Tip>
  A `time` value between `0.2` and `0.4` seconds covers most character locomotion transitions naturally. Start at `0.25` and adjust by feel.
</Tip>
