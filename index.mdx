EZAnim is a Luau module that gives you a single, consistent interface for every animation need in your Roblox game. Instead of managing raw `AnimationTrack` objects scattered across scripts, you register your animations once and let EZAnim handle playback, blending, queuing, state transitions, and ragdoll physics — all through a handful of straightforward method calls.

## Get started in three steps

Follow the steps below to go from a blank project to playing your first animation.

<Steps>
  <Step title="Require EZAnim">
    Place `EZAnim.luau` inside `ReplicatedStorage` (or `ServerScriptService` for server-only use), then require it at the top of your script.

    ```lua
    local EZAnim = require(game.ReplicatedStorage.EZAnim)
    ```
  </Step>
  <Step title="Initialize with your animation list">
    Call `EZAnim.init` and pass a `List` table that maps human-readable keys to Roblox asset IDs. EZAnim stores these internally so every subsequent call can reference animations by name.

    ```lua
    local Anim = EZAnim.init({
        List = {
            Idle = "rbxassetid://1234567890",
            Run  = "rbxassetid://0987654321",
            Jump = "rbxassetid://1122334455",
        }
    })
    ```
  </Step>
  <Step title="Play your first animation">
    Call `Anim:play` with a character model, the animation key, a priority level, whether it loops, and an optional fade time. EZAnim loads the track, applies it to the character's `Humanoid`, and returns the live `AnimationTrack`.

    ```lua
    local character = game.Players.LocalPlayer.Character
    Anim:play(character, "Idle", "Idle", true, 0.15)
    ```
  </Step>
</Steps>
