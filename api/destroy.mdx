---
title: "AnimationService:destroy() — Clean Up a Model"
sidebarTitle: "Destroy"
description: "Stop all animation tracks, disconnect every event connection, and clear all internal state for a model, fully releasing its resources from EZAnim."
---

`AnimationService:destroy()` performs a complete teardown of everything EZAnim is tracking for a given model. You should call it whenever a model is permanently removed from your game — such as when a player leaves or an NPC is despawned — to prevent memory leaks and stale event listeners from accumulating over a session.

## Signature

```lua
AnimationService:destroy(model: Model)
```

## Parameters

<ParamField path="model" type="Model" required>
  The character or NPC model to clean up. EZAnim will stop and destroy all active animation tracks for this model, disconnect all event connections (including `onMarker` and `onStopped` listeners), and clear all internal state: tracks, animators, animation cache, connections, states, queues, and ragdoll data.
</ParamField>

## Returns

This method returns `nil`.

## Example

```lua
-- When a character leaves the game
game.Players.PlayerRemoving:Connect(function(player)
    local char = player.Character
    if char then
        Anim:destroy(char)
    end
end)
```

<Note>
  EZAnim automatically calls its internal cleanup when a model's parent is set to `nil` via the `AncestryChanged` event. Even so, calling `:destroy()` manually is recommended whenever you know a model is done — it ensures cleanup happens immediately and deterministically rather than waiting for the event to fire.
</Note>

<Warning>
  After calling `:destroy()` for a model, do not attempt to call other EZAnim methods (such as `:play()` or `:stop()`) on that same model. All internal state for it has been cleared, and the results will be undefined.
</Warning>
